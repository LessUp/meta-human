## 🚀 新功能

### 功能

| 功能 | 描述 |
|------|------|
| **3D 数字人引擎** | Three.js 实时渲染，情绪驱动表情，骨骼动画 |
| **语音交互** | Web Speech API 语音合成和识别 |
| **视觉感知** | MediaPipe 人脸网格和姿态估计，情绪/手势检测 |
| **对话系统** | 支持流式（SSE）和 WebSocket 的 OpenAI 兼容对话 |
| **设备适配** | 自动硬件等级检测，最优渲染 |
| **传输抽象** | HTTP/SSE/WebSocket 无缝切换 |
| **性能指标** | 实时 FPS 跟踪和 API 响应计时 |
| **中止控制** | 取消进行中的对话请求 |

### 架构

- **状态管理：** 三个聚焦的 Zustand store
  - `chatSessionStore` 消息历史
  - `systemStore` 连接/性能指标
  - `digitalHumanStore` 数字人运行时状态
- **传输层：** 自动选择的统一接口
- **设备能力检测：** 基于硬件等级的自适应渲染

### 性能

- 低端设备跳帧处理
- 基于可见性的动画暂停
- 自适应 DPR、阴影、粒子数
- 渲染性能跟踪（FPS 指标）

## 📦 升级说明

从预发布版本升级：

### 1. 更新状态 Store 导入

```typescript
// 之前
const chatHistory = useDigitalHumanStore((s) => s.chatHistory);

// 之后
import { useChatSessionStore } from '@/store/chatSessionStore';
const chatHistory = useChatSessionStore((s) => s.chatHistory);
```

### 2. 使用新的日志

```typescript
// 之前
console.log('message');

// 之后
import { loggers } from '@/lib/logger';
const logger = loggers.app;
logger.info('message');
```

### 3. 配置传输（可选）

```bash
# .env.local
VITE_CHAT_TRANSPORT=auto
```

## 🏗️ 变更

### 代码质量

- 合并重复的可见性处理器为 `useIsTabVisibleRef` hook
- 提取 `rotateCameraHorizontal` 相机控制辅助函数
- 创建类型化常量 `TRANSPORT_LABELS` 和 `CONNECTION_STATUS_TEXT`
- 用结构化日志替换 `console.*` 调用
- 为键盘处理添加 `useCallback` 优化

### 对话系统

- 添加 abort controller 支持请求取消
- 改进流式响应的中断清理
- 增强错误处理和优雅降级
- 统一对话轮次准备逻辑

### 音频服务

- 增强 `dispose()` 清理防止内存泄漏
- ASR 服务添加预设定时器清理
- 改进回调的代际追踪

## 📊 迁移指南

### 从 v0.9.x

状态 store 域已拆分：

| 之前 | 之后 |
|------|------|
| `useDigitalHumanStore().chatHistory` | `useChatSessionStore().chatHistory` |
| `useDigitalHumanStore().connectionStatus` | `useSystemStore().connectionStatus` |

### 从 v0.8.x

无破坏性变更。流式改进向后兼容。

### 从 v0.7.x

语音服务已重构：

```typescript
// 导入路径更改
// 之前
import { ttsService } from '@core/tts';
import { asrService } from '@core/asr';

// 之后
import { ttsService, asrService } from '@core/audio';
```
