## 🚀 What's New

### Features

| Feature | Description |
|---------|-------------|
| **3D Avatar Engine** | Real-time Three.js rendering with emotion-driven expressions and skeletal animations |
| **Voice Interaction** | TTS synthesis and ASR recognition via Web Speech API |
| **Visual Perception** | MediaPipe face mesh and pose estimation for emotion/gesture detection |
| **Dialogue System** | OpenAI-compatible chat with streaming (SSE) and WebSocket support |
| **Device Adaptation** | Automatic hardware tier detection for optimal rendering |
| **Transport Abstraction** | Seamless switching between HTTP/SSE/WebSocket |
| **Performance Metrics** | Real-time FPS tracking and API response timing |
| **Abort Control** | Cancel in-progress dialogue requests |

### Architecture

- **State Management:** Three focused Zustand stores
  - `chatSessionStore` for message history
  - `systemStore` for connection/performance metrics
  - `digitalHumanStore` for avatar runtime state
- **Transport Layer:** Unified interface with auto-selection
- **Device Capability Detection:** Adaptive rendering based on hardware tier

### Performance

- Frame skipping for low-end devices
- Visibility-based animation pausing
- Adaptive DPR, shadows, and particle counts
- Render performance tracking (FPS metrics)

## 📦 Upgrade Notes

If upgrading from pre-release versions:

### 1. Update State Store Imports

```typescript
// Before
const chatHistory = useDigitalHumanStore((s) => s.chatHistory);

// After
import { useChatSessionStore } from '@/store/chatSessionStore';
const chatHistory = useChatSessionStore((s) => s.chatHistory);
```

### 2. Use New Logger

```typescript
// Before
console.log('message');

// After
import { loggers } from '@/lib/logger';
const logger = loggers.app;
logger.info('message');
```

### 3. Configure Transport (Optional)

```bash
# .env.local
VITE_CHAT_TRANSPORT=auto
```

## 📊 Migration Guide

### From v0.9.x

State store domain has been split:

| Before | After |
|--------|-------|
| `useDigitalHumanStore().chatHistory` | `useChatSessionStore().chatHistory` |
| `useDigitalHumanStore().connectionStatus` | `useSystemStore().connectionStatus` |

### From v0.8.x

No breaking changes. Streaming improvements are backward compatible.

### From v0.7.x

Voice services have been refactored:

```typescript
// Import paths changed
// Before
import { ttsService } from '@core/tts';
import { asrService } from '@core/asr';

// After
import { ttsService, asrService } from '@core/audio';
```
